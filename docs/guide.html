01项目概述

CitationClaw 是一款智能的论文被引画像分析工具。它能够自动抓取指定论文的所有施引文献，通过 LLM 分析引用学者的学术背景（院士、Fellow、企业大佬等），最终生成一份精美的可视化 HTML 画像报告。

💡 核心功能

输入论文题目或 Google Scholar 链接，一键获得完整的被引分析报告，包括学者分布、机构统计、引用趋势和主题分析。

工作流程

阶段 名称 说明
Phase 1 引用抓取 使用 ScraperAPI 从 Google Scholar 抓取所有施引论文
Phase 2 作者搜索 使用 LLM 搜索每篇施引论文作者的学术信息
Phase 3 结果导出 生成 Excel 和 JSON 格式的结构化数据
Phase 4 引用描述 （可选）分析引用论文对目标论文的引用描述
Phase 5 画像报告 生成可视化 HTML 报告，包含图表和统计

02快速开始

安装

# 通过 pip 安装
pip install citationclaw

# 启动应用
citationclaw

首次使用流程

  1. 启动后浏览器自动打开 http://127.0.0.1:8000
  2. 配置 API Key（见下方详细说明）
  3. 输入论文标题或粘贴 Google Scholar 引用链接
  4. 点击「开始分析」等待任务完成
  5. 下载生成的 Excel、JSON 和 HTML 报告

03API 配置

scraper_api_keys 必填
用于抓取 Google Scholar 的 ScraperAPI Key。免费账户有 1000 积分试用，约可爬取 50 次。
获取方式
1. 访问 https://dashboard.scraperapi.com 2. 注册账户即可获得免费 API Key 3. 复制 Key 粘贴到配置中
openai_api_key 必填
LLM API Key，支持任意 OpenAI 兼容格式的 API。推荐使用 V-API（国内可用）。
推荐配置
Base URL: https://api.gpt.ge/v1/ Search Model: gemini-3-flash-preview-search Dashboard Model: gemini-3-flash-preview-nothinking
openai_base_url 选填
API 服务地址。默认使用 V-API，如需使用其他服务请修改。
https://api.gpt.ge/v1/
openai_model 选填
用于作者搜索的模型。必须具备实时 web search 能力。
gemini-3-flash-preview-search
dashboard_model 选填
专门用于生成报告画像的模型，无需 search 能力，只需快速响应。
gemini-3-flash-preview-nothinking
⚠️ 重要提示

作者搜索模型必须具备实时 web search 能力，否则无法获取作者最新信息。Dashboard 模型可以选用更快的版本。

04爬虫配置

sleep_between_pages 选填
每页抓取之间的间隔时间（秒）。过短可能被 Google Scholar 限制。
默认值: 10 建议范围: 5-20
retry_max_attempts 选填
HTTP 请求失败时的最大重试次数。设为 -1 表示无限重试。
默认值: 3
retry_intervals 选填
重试间隔时间，逗号分隔的秒数。
默认值: "5,10,20" 含义: 第1次重试等5秒，第2次等10秒，第3次等20秒
enable_year_traverse 选填
是否按年份分别抓取。开启后能获得更完整的引用列表，但会显著增加抓取时间。
默认值: false 建议: 引用数 >1000 的论文开启
scraper_premium / scraper_ultra_premium 选填
是否使用 ScraperAPI 的高级代理。需要付费账户支持。
默认值: false
scraper_geo_rotate 选填
数据中心不一致时是否切换国家代码重试。
默认值: false 建议: 遇到数据中心不一致错误时开启

05搜索配置

parallel_author_search 选填
并行搜索作者信息的数量。增加可加速但会增加 API 消耗。
默认值: 10 建议范围: 5-20
sleep_between_authors 选填
每个作者搜索请求之间的间隔（秒）。防止 API 速率限制。
默认值: 0.5
enable_renowned_scholar_filter 选填
是否启用知名学者二次筛选。开启后 LLM 会进一步筛选出院士、Fellow 等级别学者。
默认值: true
enable_author_verification 选填
是否启用作者信息验证。会额外调用 LLM 验证作者头衔的真实性。
默认值: false 注意: 开启会显著增加 API 消耗
author_search_prompt1 / author_search_prompt2 选填
自定义作者搜索的 Prompt。一般使用默认即可。
prompt1: 搜索作者列表 prompt2: 搜索作者详细信息

06筛选配置

enable_citing_description 选填
是否搜索引用描述（Phase 4）。分析每篇引用论文如何描述目标论文。
默认值: true
citing_description_scope 选填
引用描述搜索范围。
可选值: - "all": 所有论文（默认） - "renowned_only": 仅知名学者论文 - "specified_only": 仅指定学者论文
specified_scholars 选填
当 scope 为 specified_only 时，指定要关注的学者名单（逗号分隔）。
"Geoffrey Hinton, Yann LeCun, Yoshua Bengio"
skip_author_search 选填
是否跳过作者搜索（Phase 2）。仅抓取引用列表，节省 API 费用。
默认值: false 适用场景: 快速查看引用数量

07输出配置

enable_dashboard 选填
是否生成 HTML 画像报告（Phase 5）。包含可视化图表和统计。
默认值: true
dashboard_skip_citing_analysis 选填
报告中是否跳过引用描述分析部分。
默认值: false
debug_mode 选填
调试模式。开启后会保存原始 HTML 和详细日志。
默认值: false 用途: 排查抓取问题
test_mode 选填
测试模式。使用伪造数据，不调用真实 API，用于演示或测试报告样式。
默认值: false

08报告解读

生成的 HTML 画像报告包含以下核心模块：

📊

统计概览栏

展示总引用数、施引论文数、学者数量、机构数量等关键指标。

👥

学者画像卡片

列出所有识别出的知名学者（院士、Fellow、企业大佬），显示姓名、机构、头衔和施引论文。

🏛️

机构分布卡片

展示施引学者的机构分布，分类为：国际科技企业、国内科技企业、海外顶尖高校、国内顶尖高校/机构。

📈

引用趋势图表

年度引用柱状图和月度趋势折线图，帮助了解论文影响力的变化趋势。

☁️

词云可视化

基于施引论文标题的关键词词云，直观展示引用该论文的研究主题分布。

📝

引用描述列表

（如启用 Phase 4）每篇引用论文对目标论文的引用描述，可点击展开查看详情。

输出文件说明

文件名 格式 用途
*_results.xlsx Excel 完整结果表格，包含所有字段
*_all_renowned_scholar.xlsx Excel 所有知名学者列表
*_top-tier_scholar.xlsx Excel 顶尖学者（院士级别）
*_results.json JSON 结构化数据，供程序处理
*_dashboard.html HTML 可视化画像报告（可离线查看）

09常见问题

Q1: 抓取被 Google Scholar 限制怎么办？

尝试以下方法：1) 增加 sleep_between_pages 间隔；2) 开启 scraper_premium；3) 开启 scraper_geo_rotate 切换数据中心。

Q2: 如何节省 API 费用？

1) 合理设置 parallel_author_search；2) 不需要知名学者筛选时关闭 enable_renowned_scholar_filter；3) 使用缓存复用已有结果；4) 不需要详细报告时关闭 enable_citing_description。

Q3: 作者信息不准确怎么办？

开启 enable_author_verification 进行验证，或手动检查 Searched Author Information 字段。也可尝试更换搜索模型为更强的版本（如 gemini-3-pro）。

Q4: 支持批量分析多篇论文吗？

支持！在输入框中每行输入一篇论文标题，系统会自动合并分析。支持为论文添加曾用名（别名），格式为：正式标题 | 别名1, 别名2。

Q5: 数据隐私如何保障？

所有数据保存在本地 data/ 目录，不会上传到任何服务器。API Key 仅用于调用第三方服务，建议定期更换。
